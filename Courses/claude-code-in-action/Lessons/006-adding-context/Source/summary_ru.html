При работе с Claude над кодовыми проектами управление контекстом имеет решающее значение. В проекте могут быть десятки или сотни файлов, но Claude нужна только нужная информация, чтобы помочь вам эффективно. Слишком большой объём нерелевантного контекста на самом деле снижает эффективность Claude, поэтому важно направлять его к нужным файлам и документации.

Команда /init

Когда вы впервые запускаете Claude в новом проекте, выполните команду /init. Она заставит Claude проанализировать всю кодовую базу и понять:

  • назначение и архитектуру проекта
  • важные команды и критические файлы
  • паттерны и структуру кода

После анализа кода Claude создаёт сводку и записывает её в файл CLAUDE.md. Когда Claude попросит разрешение создать этот файл, вы можете либо нажать Enter, чтобы подтверждать каждую операцию записи, либо нажать Shift+Tab, чтобы позволить Claude свободно записывать файлы в течение всей сессии.

Файл CLAUDE.md

Файл CLAUDE.md служит двум основным целям:

  • направляет Claude по вашей кодовой базе, указывая на важные команды, архитектуру и стиль кода
  • позволяет задавать Claude конкретные или пользовательские указания

Этот файл включается в каждый запрос к Claude, так что он работает как постоянный системный промпт для вашего проекта.

Расположение файлов CLAUDE.md

Claude распознаёт три разных файла CLAUDE.md в трёх распространённых местах:

  • CLAUDE.md - создаётся через /init, коммитится в исходный код и доступен другим инженерам
  • CLAUDE.local.md - не коммитится и обычно не передаётся другим инженерам; содержит личные инструкции и настройки для Claude
  • ~/.claude/CLAUDE.md - используется во всех проектах на вашей машине и содержит инструкции, которые должны применяться ко всем проектам

Добавление пользовательских инструкций

Вы можете настроить поведение Claude, добавив инструкции в файл CLAUDE.md. Например, если Claude слишком часто добавляет комментарии в код, это можно исправить через обновление файла.

Используйте команду #, чтобы войти в «режим памяти» - так вы сможете интеллектуально редактировать файлы CLAUDE.md. Просто введите что-то вроде:

# Используй комментарии экономно. Комментируй только сложный код.

Claude автоматически объединит это указание с содержимым вашего файла CLAUDE.md.

Упоминание файлов через '@'

Когда вам нужно, чтобы Claude посмотрел на конкретные файлы, используйте символ @ и укажите путь к файлу. Это автоматически включит содержимое файла в ваш запрос к Claude.

Например, если вы хотите спросить про систему аутентификации и знаете нужные файлы, можно написать:

Как работает система аутентификации? @auth

Claude покажет список связанных с аутентификацией файлов, после чего включит выбранный файл в разговор.

Ссылки на файлы в CLAUDE.md

Вы также можете упоминать файлы прямо в файле CLAUDE.md, используя ту же нотацию @. Это особенно полезно для файлов, которые важны во многих частях проекта.

Например, если у вас есть файл схемы базы данных, который определяет структуру данных, можно добавить в CLAUDE.md такую запись:

Схема базы данных определена в файле @prisma/schema.prisma. Ссылайтесь на него всякий раз, когда нужно понять структуру данных, хранящихся в базе.

Когда вы упоминаете файл таким образом, его содержимое автоматически включается в каждый ваш запрос, и Claude сможет сразу отвечать на вопросы о структуре данных без повторного поиска и чтения схемы.
