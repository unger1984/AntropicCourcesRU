Claude Code SDK позволяет запускать Claude Code программно из ваших собственных приложений и скриптов. Он доступен для TypeScript, Python и через CLI, давая те же возможности Claude Code, которыми вы пользуетесь в терминале, но уже встроенными в более крупные workflow.

SDK запускает ровно тот же Claude Code, с которым вы уже знакомы. У него есть доступ ко всем тем же инструментам, и он использует их для выполнения любой задачи, которую вы ему дадите. Это делает SDK особенно мощным для автоматизации и сценариев интеграции.

Ключевые возможности

  • Программный запуск Claude Code
  • Та же функциональность Claude Code, что и в терминале
  • Наследует все настройки от экземпляров Claude Code в той же директории
  • По умолчанию работает только с правами на чтение
  • Наиболее полезен как часть более крупных пайплайнов и инструментов

Базовое использование

Вот простой пример на TypeScript, который просит Claude проанализировать код и найти дублирующиеся запросы:

import { query } from "@anthropic-ai/claude-code";

const prompt = "Look for duplicate queries in the ./src/queries dir";

for await (const message of query({
  prompt,
})) {
  console.log(JSON.stringify(message, null, 2));
}

Когда вы запускаете этот код, вы видите «сырую» переписку между вашим локальным Claude Code и языковой моделью Claude, сообщение за сообщением. Финальное сообщение содержит полный ответ Claude.

Разрешения и инструменты

По умолчанию SDK имеет только права на чтение. Он может читать файлы, искать по директориям и выполнять grep, но не может писать, редактировать или создавать файлы.

Чтобы разрешить запись, можно добавить в запрос опцию allowedTools:

for await (const message of query({
  prompt,
  options: {
    allowedTools: ["Edit"]
  }
})) {
  console.log(JSON.stringify(message, null, 2));
}

Альтернативно можно настроить разрешения в файле настроек внутри директории .claude и тем самым выдать доступ на уровне всего проекта.

Практические применения

Claude Code SDK особенно хорош, когда его встраивают в более крупные инженерные workflow. Его можно использовать для:

  • Git hooks, которые автоматически проверяют изменения кода
  • Скриптов сборки, анализирующих и оптимизирующих код
  • Вспомогательных команд для задач сопровождения кода
  • Автоматической генерации документации
  • Проверок качества кода в CI/CD-пайплайнах

По сути, SDK позволяет добавить ИИ-интеллект в любую часть вашего процесса разработки, где программный доступ действительно полезен.
