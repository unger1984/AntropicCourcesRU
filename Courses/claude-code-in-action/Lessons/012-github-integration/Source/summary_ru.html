Claude Code предлагает официальную интеграцию с GitHub, которая позволяет Claude работать внутри GitHub Actions. У этой интеграции два основных сценария: поддержка упоминаний в issues и pull request, а также автоматические обзоры pull request.

Настройка интеграции

Чтобы начать, запустите в Claude команду /install-github-app. Она проведет вас через несколько шагов:

  • Установить приложение Claude Code в GitHub
  • Добавить API-ключ
  • Автоматически создать pull request с файлами workflow

Созданный pull request добавляет в репозиторий два GitHub Actions. После слияния файлы workflow окажутся в каталоге .github/workflows.

Стандартные действия GitHub

Интеграция предлагает два основных workflow:

Действие упоминания

В любом issue или pull request можно упомянуть Claude с помощью @claude. После упоминания Claude:

  • Проанализирует запрос и составит план задачи
  • Выполнит задачу с полным доступом к вашей кодовой базе
  • Ответит с результатом прямо в issue или pull request

Действие pull request

Когда вы создаете pull request, Claude автоматически:

  • Проверяет предложенные изменения
  • Анализирует влияние модификаций
  • Публикует подробный отчет в pull request

Настройка workflow

После слияния первоначального pull request вы можете настроить файлы workflow под нужды своего проекта. Вот как можно расширить workflow для упоминаний:

Добавление настройки проекта

Перед запуском Claude можно добавить шаги подготовки окружения:

- name: Project Setup
  run: |
    npm run setup
    npm run dev:daemon

Пользовательские инструкции

Передайте Claude контекст о настройке проекта:

custom_instructions: |
  The project is already set up with all dependencies installed.
  The server is already running at localhost:3000. Logs from it
  are being written to logs.txt. If needed, you can query the
  db with the 'sqlite3' cli. If needed, use the mcp__playwright
  set of tools to launch a browser and interact with the app.

Конфигурация MCP-сервера

Можно настроить MCP-серверы, чтобы дать Claude дополнительные возможности:

mcp_config: |
  {
    "mcpServers": {
      "playwright": {
        "command": "npx",
        "args": [
          "@playwright/mcp@latest",
          "--allowed-origins",
          "localhost:3000;cdn.tailwindcss.com;esm.sh"
        ]
      }
    }
  }

Права доступа к инструментам

При запуске Claude в GitHub Actions нужно явно перечислить все разрешенные инструменты. Это особенно важно при использовании MCP-серверов.

allowed_tools: "Bash(npm:*),Bash(sqlite3:*),mcp__playwright__browser_snapshot,mcp__playwright__browser_click,..."

В отличие от локальной разработки, в GitHub Actions нет короткого пути для прав доступа. Каждый инструмент из каждого MCP-сервера нужно перечислять отдельно.

Лучшие практики

При настройке GitHub-интеграции Claude:

  • Начинайте со стандартных workflow и настраивайте их постепенно
  • Используйте пользовательские инструкции, чтобы передать контекст проекта
  • Явно указывайте права доступа к инструментам при использовании MCP-серверов
  • Проверяйте workflow на простых задачах до перехода к сложным
  • Учитывайте конкретные потребности проекта при добавлении дополнительных шагов

GitHub-интеграция превращает Claude из помощника разработчика в автоматического члена команды, который может выполнять задачи, проверять код и давать рекомендации прямо внутри вашего GitHub-процесса.
