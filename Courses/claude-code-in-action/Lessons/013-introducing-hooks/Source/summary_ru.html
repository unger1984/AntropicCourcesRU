Hooks позволяют запускать команды до или после того, как Claude пытается выполнить инструмент. Это очень полезно для автоматизации рабочих процессов, например для запуска форматтеров после редактирования файлов, выполнения тестов при изменении файлов или ограничения доступа к определенным файлам.

Как работают hooks

Чтобы понять hooks, сначала вспомним обычный поток работы, когда вы взаимодействуете с Claude Code. Когда вы задаете Claude вопрос, ваш запрос отправляется модели Claude вместе с определениями инструментов. Затем Claude может решить использовать инструмент, сформировав ответ в нужном формате, а Claude Code выполнит этот инструмент и вернет результат.

Hooks встраиваются в этот процесс и позволяют запускать код прямо перед выполнением инструмента или сразу после него.

Есть два типа hooks:

  • PreToolUse hooks - запускаются до вызова инструмента
  • PostToolUse hooks - запускаются после вызова инструмента

Настройка hooks

Hooks задаются в файлах настроек Claude. Их можно добавить в:

  • Global - ~/.claude/settings.json (действует для всех проектов)
  • Project - .claude/settings.json (общий для команды)
  • Project (not committed) - .claude/settings.local.json (личные настройки)

Вы можете писать hooks вручную в этих файлах или использовать встроенную команду /hooks прямо внутри Claude Code.

Сама конфигурация выглядит примерно так, как показано справа на экране. Давайте разберем пример файла, чтобы лучше понять, что здесь происходит.

Сначала обратите внимание, что в этом файле есть две отдельные секции. Одна секция перечисляет все команды, которые нужно выполнить до использования инструмента. Это и есть hooks типа pre-tool use. Другая секция перечисляет команды, которые должны выполняться после использования инструмента. Это hooks типа post-tool use.

В каждой из этих секций мы задаем matcher. Он указывает, какие именно типы вызовов инструментов нас интересуют. В этом примере мне нужны вызовы инструмента read. Каждый раз, когда Claude Code пытается прочитать файл, я хочу запускать показанную там команду. Аналогично, в секции post tool use после вызова write, edit или multi-edit запускается другая команда.

Вот что hooks действительно должны делать. Эти команды получают сведения о вызове инструмента, который хочет выполнить Claude. В случае pre-tool use hook вы можете проверить, что именно хочет сделать Claude. И если по какой-то причине вы не хотите это разрешать, можно заблокировать вызов инструмента и отправить Claude сообщение об ошибке. В случае post tool use hook вызов инструмента уже произошел, так что блокировать его поздно. Но вы можете выполнить последующее действие на основе этого вызова, например отформатировать файл, который только что был отредактирован.

Вы также можете отправить Claude дополнительное сообщение о самом использовании инструмента. Например, вы можете запустить отдельную программу, чтобы проверить качество кода в изменении, или выполнить type check, а затем вернуть этот результат Claude. После этого Claude может учесть обратную связь и обновить файл, который он только что записал.

Если hooks или их назначение все еще кажутся вам непонятными, это совершенно нормально. Разобраться в hooks может быть действительно непросто. Поэтому давайте на время отложим теорию и поработаем на примере проекта с hooks.

Применение на практике

Вот несколько распространенных способов использовать hooks:

  • Code formatting - автоматически форматировать файлы после редактирования Claude
  • Testing - запускать тесты автоматически при изменении файлов
  • Access control - блокировать Claude от чтения или редактирования определенных файлов
  • Code quality - запускать линтеры или type checker и передавать Claude обратную связь
  • Logging - отслеживать, какие файлы Claude читает или изменяет
  • Validation - проверять соглашения об именовании или стандарты кода

Главная идея в том, что hooks позволяют расширять возможности Claude Code, встраивая в рабочий процесс ваши собственные инструменты и процессы. PreToolUse hooks дают вам контроль над тем, что Claude может делать, а PostToolUse hooks позволяют усиливать то, что Claude уже сделал.
