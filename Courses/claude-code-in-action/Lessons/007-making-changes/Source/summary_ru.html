Когда вы работаете с Claude в своей среде разработки, вам часто нужно вносить изменения в существующие проекты. Это руководство показывает практические приёмы для эффективного внесения изменений, включая визуальную коммуникацию со скриншотами и использование расширенных возможностей рассуждения Claude.

Использование скриншотов для точной коммуникации

Один из самых эффективных способов общаться с Claude - использовать скриншоты. Когда вы хотите изменить конкретную часть интерфейса, скриншот помогает Claude точно понять, о чём идёт речь.

Чтобы вставить скриншот в Claude, используйте Ctrl+V (а не Cmd+V на macOS). Эта комбинация специально предназначена для вставки скриншотов в чат. После вставки изображения вы можете попросить Claude внести точечные изменения в нужную область приложения.

Режим планирования

Для более сложных задач, которые требуют серьёзного исследования по всей кодовой базе, можно включить режим планирования. Эта функция заставляет Claude сначала тщательно исследовать проект, а уже потом вносить изменения.

Режим планирования включается нажатием Shift + Tab два раза (или один раз, если у вас уже включено автоматическое принятие правок). В этом режиме Claude будет:

  • читать больше файлов в проекте
  • создавать подробный план реализации
  • показывать, что именно он собирается сделать
  • ждать вашего подтверждения перед продолжением

Так у вас будет возможность проверить план и скорректировать Claude, если он что-то упустил или не учёл какой-то сценарий.

Режимы мышления

Claude предлагает разные уровни рассуждения через режимы «мышления». Они позволяют Claude потратить больше времени на обдумывание сложных задач перед тем, как дать решение.

Доступные режимы мышления включают:

  • «Think» - базовое рассуждение
  • «Think more» - расширенное рассуждение
  • «Think a lot» - всестороннее рассуждение
  • «Think longer» - более длительное рассуждение
  • «Ultrathink» - максимальная глубина рассуждения

Каждый следующий режим даёт Claude всё больший бюджет токенов для работы, что позволяет глубже анализировать сложные задачи.

Когда использовать планирование и когда мышление

Эти две функции решают разные задачи по сложности:

Режим планирования лучше всего подходит для:

  • задач, требующих широкого понимания кодовой базы
  • многошаговых реализаций
  • изменений, затрагивающих несколько файлов или компонентов

Режим мышления лучше всего подходит для:

  • сложных логических задач
  • отладки трудных ошибок
  • алгоритмических задач

Эти режимы можно сочетать, если задача требует и широты, и глубины. Только помните, что оба режима расходуют дополнительные токены, поэтому при их использовании есть связанная с этим стоимость.
