Hooks в Claude Code позволяют перехватывать и управлять вызовами инструментов до или после их выполнения. Это даёт вам тонкий контроль над тем, что Claude может и чего не может делать в вашей среде разработки.

Создание hook’а

Создание hook’а состоит из четырёх основных шагов:

  1. Выберите PreToolUse или PostToolUse hook - PreToolUse hooks могут не дать вызову инструмента выполниться, а PostToolUse hooks запускаются после того, как инструмент уже был использован
  2. Определите, за какими именно вызовами инструментов вы хотите следить - нужно точно указать, какие инструменты должны запускать ваш hook
  3. Напишите команду, которая будет получать вызов инструмента - эта команда получает JSON-данные о предлагаемом вызове через стандартный ввод
  4. Если нужно, команда должна вернуть обратную связь Claude - код выхода вашей команды сообщает Claude, разрешить операцию или заблокировать её

Структура данных вызова инструмента

Когда ваша команда hook’а выполняется, Claude передаёт JSON-данные через стандартный ввод, содержащие сведения о предлагаемом вызове инструмента:

{
  "session_id": "2d6a1e4d-6...",
  "transcript_path": "/Users/sg/...",
  "hook_event_name": "PreToolUse",
  "tool_name": "Read",
  "tool_input": {
    "file_path": "/code/queries/.env"
  }
}

Ваша команда читает этот JSON из стандартного ввода, разбирает его, а затем решает, разрешить операцию или заблокировать её, исходя из имени инструмента и параметров входных данных.

Коды выхода и поток управления

Ваша hook-команда сообщает Claude через коды выхода:

  • Код выхода 0 - всё в порядке, разрешить вызов инструмента
  • Код выхода 2 - заблокировать вызов инструмента (только для PreToolUse hooks)

Если в PreToolUse hook вы завершаетесь с кодом 2, любые сообщения об ошибке, записанные в стандартный поток ошибок, также будут отправлены Claude как обратная связь с объяснением, почему операция была заблокирована.

Пример использования

Типичный сценарий - запретить Claude читать чувствительные файлы, например файлы .env. Поскольку и инструменты Read, и Grep могут получать доступ к содержимому файлов, нужно отслеживать оба типа инструментов и проверять, не пытаются ли они обращаться к защищённым путям.

Такой подход даёт вам полный контроль над доступом Claude к файловой системе, а также понятную обратную связь о том, почему некоторые операции ограничены.
