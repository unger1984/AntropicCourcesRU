Claude Code поставляется со встроенными командами, доступными через слэш, но вы также можете создавать собственные команды, чтобы автоматизировать повторяющиеся задачи, которые выполняете часто.

Создание пользовательских команд

Чтобы создать пользовательскую команду, нужно настроить в проекте определённую структуру папок:

  1. Найдите папку .claude в каталоге проекта
  2. Создайте внутри неё новый каталог commands
  3. Создайте новый markdown-файл с нужным именем команды, например audit.md

Имя файла становится именем команды, так что audit.md создаёт команду /audit.

Эта команда audit делает три вещи:

  1. Запускает npm audit, чтобы найти уязвимые установленные пакеты
  2. Запускает npm audit fix, чтобы применить обновления
  3. Запускает тесты, чтобы убедиться, что обновления ничего не сломали

После создания файла команды нужно перезапустить Claude Code, чтобы он распознал новую команду.

Команды с аргументами

Пользовательские команды могут принимать аргументы через заполнитель $ARGUMENTS. Это делает их гораздо гибче и удобнее для повторного использования.

Например, команда write_tests.md может содержать:

Напишите полный набор тестов для: $ARGUMENTS

Правила тестирования:
* Используйте Vitest с React Testing Library
* Размещайте файлы тестов в каталоге __tests__ в той же папке, что и исходный файл
* Называйте файлы тестов как [filename].test.ts(x)
* Используйте префикс @/ для импортов

Покрытие:
* Проверяйте успешные сценарии
* Проверяйте граничные случаи
* Проверяйте сценарии ошибок

Затем вы можете запустить эту команду с путём к файлу:

/write_tests файл use-auth.ts в каталоге hooks 

Аргументы не обязаны быть путями к файлам — это может быть любая строка, которую вы хотите передать Claude для контекста и направления задачи.

Ключевые преимущества

  • Автоматизация — превращайте повторяющиеся рабочие процессы в одну команду
  • Последовательность — обеспечивайте одинаковое выполнение шагов каждый раз
  • Контекст — задавайте Claude конкретные инструкции и соглашения для вашего проекта
  • Гибкость — используйте аргументы, чтобы команды работали с разными входными данными

Пользовательские команды особенно полезны для специфичных для проекта процессов, таких как запуск тестовых наборов, развёртывание кода или генерация шаблонов по соглашениям вашей команды.
